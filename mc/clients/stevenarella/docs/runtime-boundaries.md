# Stevenarella runtime-boundary audit

This audit records the invariants for the runtime-boundary hardening change. It is not a full memory-safety proof and does not broaden Minecraft compatibility, renderer portability, public-server safety, production readiness, or full CTF/survival correctness claims.

## Boundary inventory

| Boundary | Current mechanism | Caller invariant | Failure mode | Mitigation in this change |
| --- | --- | --- | --- | --- |
| GL context ownership (`src/gl/mod.rs`) | One process-wide `glow::Context` reached by all GL wrappers | Startup creates the GL context once before any renderer-owned object is constructed or dropped | Uninitialized access panics through an unchecked `unwrap`; duplicate initialization leaks/replaces context | Quarantine the global behind an atomic pointer, an explicit `GlContextToken`, deterministic initialization errors, and tests for uninitialized access |
| GL object wrappers (`Texture`, `Buffer`, `Program`, `Shader`, `VertexArray`, `Framebuffer`) | Thin safe structs call `glow` unsafe methods | A current GL context exists for the lifetime of every wrapper and destructor | Dropping after context loss or before initialization calls into an invalid context | Route context lookup through the checked GL boundary and document startup-only panic policy |
| GL slice reinterpretation (`Uniform::set_float_multi`, `set_matrix4`, `set_matrix4_multi`) | Casts cgmath data into contiguous `f32` slices | cgmath matrix/vector layout matches the contiguous shader upload expected by glow | Incorrect layout would upload invalid uniform data | Keep the unsafe cast local to the GL module; callers only pass typed slices/matrices |
| Mapped GL buffers (`Buffer::map`, `MappedBuffer`) | Converts a mapped GPU pointer into a `Vec<u8>` and forgets it on drop | The mapped pointer is valid for the requested length until unmap | Bad length or null pointer would create invalid host memory access | Leave as documented GL-only unsafe surface; do not expose raw pointers outside `gl` |
| Resource manager sharing (`src/resources.rs`) | `Manager` is shared as `Arc<RwLock<Manager>>` and had an unsafe `Sync` implementation | Pack IO is read under the manager lock; reload state mutates under the write lock; worker progress uses its own mutex | Unsafe `Sync` could hide fields that are not actually synchronized; poisoned progress lock panics | Remove unsafe `Sync`, name `SharedManager`, keep channels/reload state under `RwLock`, and route progress updates through fail-closed helpers |
| Resource pack IO | `Pack::open` returns `Option<Box<dyn Read>>` from internal, directory, or object packs | Missing resources are expected and represented as `None`; successful reads own their reader | Missing or malformed resource can panic at higher parsing layers if caller unwraps | Boundary tests cover present/missing resources; higher render/model parsing remains a non-claim |
| Resource progress state | Download workers update `Arc<Mutex<Progress>>`; UI reads and prunes tasks during ticks | Worker updates do not require the manager write lock; UI state is only touched by the main thread | Poisoned mutex used to panic and could take down unrelated render/resource code | Progress helpers return a deterministic `ResourceBoundaryError::ProgressPoisoned` and tick fails closed |
| ECS entity generation (`src/ecs/mod.rs`) | Entity IDs carry a generation and state slot | Safe access must reject stale generations, removed slots, and out-of-range IDs before touching component storage | Stale entity or absent slot could panic or read a component from a reused entity | Accessors validate generation/membership before storage access; tests cover stale generation and removed entities |
| ECS component membership | Bit sets track current/last membership; component blocks track initialized slots | Component storage may be read only when both entity membership and storage bit are present | Missing component or storage block unwraps | ComponentMem exposes checked `get`/`get_mut` methods returning `Option` |
| ECS aliasing | Existing public methods used lifetime transmute to permit overlapping borrows | Callers must not hold incompatible component borrows for the same entity/key | Safe callers could create aliased mutable references | Public accessors return lifetimes tied to the manager; representative systems copy immutable data before taking mutable access |
| ECS drop | Raw component bytes are dropped through a type-erased drop function | A component slot is dropped exactly once after membership is cleared or on storage drop | Double-remove/drop or missing slot would corrupt state | Removal checks storage membership, clears the bit, and tests verify drop counts for removal and manager drop |
| Chunk-builder/model resource use (`src/chunk_builder.rs`, `src/model/mod.rs`, `src/render/mod.rs`) | Worker threads hold `Arc<RwLock<model::Factory>>`; the factory owns shared resource/texture managers | Worker reads/mutates model cache under locks; resource manager itself is not accessed without synchronization | Lock poisoning still panics through existing model/render unwraps | Representative call sites use the named `resources::SharedManager`; broader lock-poison recovery remains a non-claim |
