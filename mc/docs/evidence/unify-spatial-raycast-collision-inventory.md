# Spatial raycast/collision integration inventory

## Question
Which Valence and Hyperion spatial/raycast/collision sources are safe to use for the `unify-spatial-raycast-collision` change?

## Decision
Port only stable concepts into Valence-owned `valence_spatial` helpers. Do not copy Hyperion nightly, unsafe-heavy, runtime-coupled, Bedwars, or production-event code. Existing Valence BVH, AABB, hitbox, chunk, and generated collision-shape APIs are adopted as integration surfaces; Hyperion sources remain reference/port inputs only.

## Boundary classification
This inventory applies `docs/hyperion-integration-boundaries.md`: all Hyperion sources are `port`, `reference`, or `reject`; none are adopted directly. Default Valence behavior remains unchanged unless callers opt into the new pure helpers.
