# replace-entity-keyed-state-with-components inventory

## Scope

Selected Valence-owned runtime state was inventoried in `servers/valence/examples/*.rs`, `servers/valence/examples/fixture_core/*.rs`, and the Rust mc-compat runner shell under `compat/runner/src/*.rs`. The implementation slice is limited to `servers/valence/examples/survival_compat.rs`; no Hyperion code or concepts were used.

## Verification intent

The migrated component has one active kind per client, so opening another fixture replaces the previous open-container state instead of creating duplicate ownership. Click handlers fail closed when the component is missing or has the wrong fixture kind. Cleanup removes the component from `Despawned` clients so reconnecting entities start without stale open-container state.
