# Proposal: Add paired survival reference parity receipts

## Summary

Add paired reference-server and Valence receipts for the existing survival break/place/pickup rail so vanilla/Paper parity can be evaluated for one narrow survival loop.

## Motivation

The current survival row proves Valence can satisfy a deterministic break/place/pickup fixture, but it does not prove vanilla parity. A paired reference receipt is needed before any parity claim can be promoted.

## Scope

- Run the same Stevenarella survival probe against a local reference server fixture.
- Record matching Valence and reference receipts with reviewable logs.
- Compare exact block update, inventory slot, pickup, and placement metrics.
- Keep full survival compatibility and broad vanilla parity as non-claims.

## Non-goals

- No full survival parity claim.
- No crafting/furnace/chest/hunger/mob/redstone coverage.
- No public-server or production-load claim.
