## Why

The sign-editor open/update row is blocked by missing maintained Stevenarella automation for observing a sign editor open and submitting a bounded four-line update through the protocol path. A client driver would unlock live sign-edit evidence without reusing sign-persistence proof or claiming UI breadth.

## What Changes

- Add a bounded Stevenarella sign-editor driver or control-plane action for one configured sign position and four-line payload.
- Record client open/update milestones, protocol update path, Valence accepted-update correlation, child revisions when available, and explicit non-claims.
- Validate positive and negative driver behavior before any targeted-packet live promotion.
- Keep sign editing UI behavior, all sign variants, arbitrary sign text/NBT semantics, all block entities, public-server safety, production readiness, and full protocol coverage as non-claims.

## Impact

- **Files**: `stevenarella/**` when implementation begins, `tools/mc-compat-runner/src/**`, `tools/check_targeted_packet_promotions.rs`, `docs/evidence/**`, and targeted packet docs if evidence later promotes the row.
- **Testing**: Stevenarella focused tests via the mc devshell, runner dry-run/unit checks, positive and negative sign-driver validation, targeted-packet live-evidence checker, Cairn gates and validation.
